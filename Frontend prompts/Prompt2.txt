Create a complete component system for a dark AI-infrastructure SaaS website and dashboard in an InsForge-like visual language.

## Global shell
- background: black
- max-width container: 1280px
- page gutters: 24px mobile, 32px tablet, 40px desktop
- borders: 1px solid dark gray
- border radius: 6px to 10px max
- use a subtle grid overlay in certain hero/feature sections
- maintain strong vertical rhythm between sections

## Header / Navbar
Build a compact, fixed or sticky top navigation bar with:
- left-aligned logo
- compact nav links
- muted text by default, white on hover
- right-aligned GitHub/login/sign-up actions
- primary CTA in emerald solid
- secondary actions as text or ghost
- use small font size with strong tracking discipline
- keep it elegant and non-bulky

## Hero section
Build a hero with:
- small credibility badge at top
- strong, large headline
- one highlighted keyword using emerald accent
- short subheading paragraph with constrained width
- primary and secondary CTA buttons
- optional central system visual / network visual / governance diagram
- subtle grid lines or framing borders
- not too many paragraphs
- should feel like infrastructure product, not marketing fluff

## Feature bento grid
Build a 12-card feature grid:
- use shared borders like a continuous grid
- mix card heights and widths
- cards should feel like system primitives
- each card must contain:
  - icon
  - feature title
  - 1-line value proposition
  - small technical visual or mini mockup
- no giant illustrations
- every card should look like a product capability tile

## Product proof / workflow section
Build a section showing how the product works:
- request enters
- identity resolved
- policy evaluated
- taint checked
- risk scored
- approval routed
- execution allowed/blocked
- audit evidence written
Represent this as a dark bordered workflow rail or horizontally connected system cards.

## Detail page sections
Every feature page should contain:
- hero
- what problem it solves
- how it works
- key capabilities
- architecture visual
- screenshots/mock cards
- example policy or code snippet
- FAQ
- CTA

## Testimonial section
Build a social proof section using:
- bordered quotes
- author name and role
- monochrome avatars or minimal placeholders
- avoid colorful cards
- keep it text-led and calm

## Pricing section
Use a precise pricing layout:
- dark cards
- clear border hierarchy
- one highlighted plan
- feature checklists with mono bullets or technical icons
- enterprise CTA card

## Footer
Build a utility footer:
- multi-column grid
- docs, product, company, legal, github
- small muted text
- monochrome social icons
- subtle top border

## Buttons
Primary:
- emerald fill
- dark text
- compact height
- medium font weight
Secondary:
- transparent or dark fill
- border
- white text

## Badges and chips
- use mono font
- small size
- subtle border
- muted text
- active state in emerald tint

## Code blocks
- use Fira Code
- dark bordered shell
- small title bar
- line-height tuned for readability
- syntax highlighting minimal and elegant

## Tables / comparison grids
- thin borders
- very readable spacing
- row hover subtle
- support feature comparison pages

## Dashboard preview widgets
When showing screenshots/mock previews for product features:
- use panelized dark cards
- strong section titles
- subtle data points
- mint accents only for signal, not decoration
- include logs, policies, traces, risk badges, audit rows, approval rows

## Animations
- fade-up on scroll
- slight border glow on hover
- optional marquee for integrations
- optional animated grid or system tunnel in hero
- keep movement technical and restrained
